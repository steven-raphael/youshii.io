Purpose

減少人工成本與紙本消耗、24小時都能接單、交接顧客需求不漏接、優化動線、特定的上游下單，讓繁雜的生活多一些時間。

Design Thingking

Design Thinking 是以使用者為中心去解決問題的方法。由於時程比較趕以及我必須協助前端切版，又是一人設計師，所以我將UX濃縮主要只做UI FLOW 來確保所有資訊架構流程

Discover

過往的資料分析

PM接收回來的資料分析，深入了解餐廳供應鏈上下游的看法及相關場景、需求體驗以及感受，有效的幫助我們強調不同的用戶群體。
於是我們列出了以下的需求問題：
減少人工成本
減少紙本消耗
24小時都能接單
訂貨交貨動線優化
交接顧客需求不漏接

Define

Value Proposition

在定義的階段，透過問題陳述、假設，更好的去歸納出使用者的痛點和需求，並提出可能的解決方案。
於是我們列出了以下的解決方案：
範本管理
訂單管理
新增訂單
公告
定義 deisgin system

Ideate

UI FLOW

協助PM製作完的 user flow、IA後，協助檢視是否有滿足所有關鍵的目標需求、功能轉換是否順暢，直接將 UI FLOW 轉換為 mockup 製作成 Hi-fi Prototype,並進行易用性測試，並檢視我們的產品是否有需要優化和疏漏的功能
