Purpose

結合物理治療APP與ERP，提供物理治療師分析數據以及上下架運動計畫等等的系統

Design Thingking

Design Thinking 是以使用者為中心去解決問題的方法。由於整個產品是由我們遇到的問題和解決問題的驅動力所激發的，因此我選擇最常使用的雙鑽石設計流程來引導我們完成整個產品設計過程。

Discover

使用者訪談

進行結構化的一對一訪談，深入了解物理治療師對於系統的看法及相關場景、需求體驗以及感受，有效的幫助我們強調不同的用戶群體。
於是我們列出了以下的需求問題：
自訂運動處方
病患情況分析
運動輔具上傳
疼痛問卷上傳
app即時訊息

User Story Map

與使用者一起走過整個旅程，可以更清楚看到他們的真實需求並為他們量身定製功能。
於是我們列出了以下的設計目標：
運動處方
處方紀錄
運動模組
運動叮嚀
線上評估紀錄
動作元件
動作模組
輔具管理
問卷管理
營運管理
帳號管理

Define

Value Proposition

在定義的階段，透過問題陳述、假設，更好的去歸納出使用者的痛點和需求，並提出可能的解決方案。
於是我們列出了以下的解決方案：
優先開發那些設計目標
敏捷還是瀑布流開發
定義 deisgin system

Ideate

Wireframe

PM製作完的 user flow 以及 IA後，協助檢視是否有滿足所有關鍵的目標需求、功能轉換是否順暢，我將 wireframe 製作成 Lo-fi Prototype,並進行易用性測試，減少後續mockup的時間以及資訊的浪費，並檢視我們的產品是否有需要優化和疏漏的功能
