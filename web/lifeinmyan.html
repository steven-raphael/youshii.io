Purpose

主要市場為東南亞地區，結合食衣住行育樂以及分享優惠券的電商網站

Design Thingking

Design Thinking 是以使用者為中心去解決問題的方法。由於時程比較趕以及我必須擔任行銷文宣 PM UIUX 前端切版的角色，所以我將UX濃縮主要兩個部分，分別為 UI FLOW 以及 Wireframe 來確保所有資訊架構流程

Discover

過往的資料分析

從主管接收回來的資料分析，深入了解東南亞那邊的電商的看法及相關場景、需求體驗以及感受，有效的幫助我們強調不同的用戶群體。
於是我們列出了以下的需求問題：
繽紛色彩東南亞風格
設計酷碰卷
結合食衣住行育樂
條件篩選功能
廣告投放
多國語系

競品分析

參考想要的風格網站,深入解析優缺點以及痛點。使我對此網站設計該如何發展有了更清晰的雛型。
於是我們列出了以下的目標問題：
旅遊區
美食區
住宿區
休閒娛樂區
達人分享
公益活動
會員註冊
商家註冊
最新優惠
新進商家
精選區

Define

Value Proposition

在定義的階段，透過問題陳述、假設，更好的去歸納出使用者的痛點和需求，並提出可能的解決方案。
於是我們列出了以下的解決方案：
優先開發那些設計目標
定義 deisgin system

Ideate

UI FLOW

製作 UI flow，協助檢視是否有滿足所有關鍵的目標需求、功能轉換是否順暢。

Wireframe

製作 wireframe 並製作成 Lo-Fi prototype,並進行易用性測試，並檢視我們的產品是否有需要優化和疏漏的功能。
