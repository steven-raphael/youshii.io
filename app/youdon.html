Design Thingking

Design Thinking 是以使用者為中心去解決問題的方法。由於整個產品是由我們遇到的問題和解決問題的驅動力所激發的，因此我選擇最常使用的雙鑽石設計流程來引導我們完成整個產品設計過程。

Discover

使用者訪談

進行結構化的一對一訪談，深入了解物理治療師以及長期復健的病患對於APP的看法及相關場景、需求體驗以及感受，有效的幫助我們強調不同的用戶群體。
於是我們列出了以下的需求問題：
物理治療師量身設計的運動計畫
了解運動執行情況與疼痛變化
隨時隨地運動
線上視訊諮詢
線下預約物理治療所

競品分析

透過競爭對手hingehealth、kaiahealth、luna、omadahealth、speedoc的app以及網站,深入解析優缺點以及痛點。使我對APP該如何發展以及我們的產品該如何讓使用者快速上手有了更清晰的雛型。
於是我們列出了以下的需求問題：
視訊諮詢
使用者疼痛問題分析
運動完的獎金回饋機制

User Story Map

與使用者一起走過整個旅程，可以更清楚看到他們的真實需求並為他們量身定製功能。
於是我們列出了以下的需求問題：
疼痛數據分析
自動調整動作
個人化運動計畫
運動統計
運動提醒
預約分院
視訊諮詢
疼痛檢測
運動數據圖表
肌肉骨骼檢測

Define

Value Proposition

在定義的階段，透過問題陳述、假設，更好的去歸納出使用者的痛點和需求，並提出可能的解決方案。
於是我們列出了以下的需求問題：
優先開發那些設計目標
敏捷還是瀑布流開發
定義 deisgin system

Ideate

Wireframe

PM製作完的 user flow 以及 IA後，協助檢視是否有滿足所有關鍵的目標需求、功能轉換是否順暢，我將 wireframe 製作成 Lo-fi Prototype,並進行易用性測試，減少後續mockup的時間以及資訊的浪費，並檢視我們的產品是否有需要優化和疏漏的功能

Design Guideline

定製 Deisgn Guideline，提供有關如何建立和實施設計的具體建議。確保設計在整個產品、品牌或項目中保持一致性和品質。
